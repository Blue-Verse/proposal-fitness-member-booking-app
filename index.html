  • 앱 내 1:1 채팅 — 영상·파일 전송 지원, 개인 메신저 대체하여 개인정보 노출 방지
  • 회원 가입·로그인·정보 관리 등 기본 회원 기능
  • 예약 확정/취소, 수업 리마인더, 채팅 수신 등 주요 푸시 알림
  • 관리자 페이지 — 회원/스케줄/영상/공지/결제/통계 통합 관리
  • 기존 외부 솔루션 + 네이버 예약 누적 회원 정보 안전 이전(마이그레이션)

제안 포인트

1. 피트니스·헬스케어 도메인 수행 경험

필라테스 프랜차이즈 관리 앱(수업 예약/강사/결제/CRM, Android·iOS·Web 동시 출시), 시니어 주간보호 관리 플랫폼(실시간 케어 로그·전자서명·AI 분석) 등 유사 도메인 프로젝트를 풀스택으로 수행했습니다. 수업 예약, 출결, 결제, 강사-회원 커뮤니케이션 흐름을 이미 다수 구축한 경험을 기반으로 초기 시행착오를 최소화합니다.

2. 크로스플랫폼(Flutter) + 관리자 웹 동시 구축

Android/iOS 앱을 단일 코드베이스(Flutter)로 구축하고, 관리자 페이지는 Next.js 기반 웹으로 구성합니다. 하나의 백엔드(NestJS)를 공유해 기능별 중복 구현을 줄이고, 스토어 심사/빌드 주기와 웹 배포 주기를 병렬로 운영합니다.

3. 개인정보 비노출형 1:1 채팅

WebSocket 기반 실시간 채팅에 영상·파일 업로드, 읽음/전송 상태, 푸시 알림까지 포함해 개인 메신저를 완전히 대체합니다. 강사와 회원의 전화번호·개인 ID가 상호 노출되지 않도록 앱 내 식별자만으로 소통하게 설계합니다.

4. 체계적인 운동 영상 콘텐츠 구조

공통 영상 라이브러리는 카테고리·부위·난이도 기반 메타데이터로 관리하고, 개인 복습 영상은 강사가 앱에서 직접 업로드해 해당 회원에게만 노출되는 권한 구조로 설계합니다. 영상은 AWS S3 + CloudFront(HLS 스트리밍) 기반 안정적 재생을 보장합니다.

5. 데이터 마이그레이션 전담 단계 편성

외부 솔루션 및 네이버 예약에서 추출한 회원·수강권·예약 데이터를 사전 매핑 스크립트로 검증합니다. 실데이터 이관 전 2회 이상의 리허설(Dry-run)을 거쳐 누락·중복·오매핑 리스크를 차단하고, 이관 실패 시 롤백 절차까지 제공합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

회원 관리, 예약, 소통 채널을 하나로 통합하는 센터 전용 모바일 앱을 구축해 운영 효율성을 극대화하고 전문적인 서비스를 제공합니다. 앱 내 공식 소통 창구를 통해 회원-강사 간 불필요한 개인 정보 노출을 방지하고, 체계적인 운동 영상 콘텐츠를 제공하여 회원 만족도를 높이는 것을 목표로 합니다.

2. 사용자 정의

사용자 유형설명주요 기능
회원센터에 등록된 회원. 앱 주 사용자수업 예약/변경/취소, 수강권 결제, 공통·개인 영상 시청, 강사와 1:1 채팅, 푸시 수신
강사센터 소속 강사. 회원과 1:1 수업 진행본인 스케줄 확인, 예약 승인/거절, 개인 복습 영상 전송, 회원 1:1 채팅
관리자센터 운영자. 전체 운영 관리회원/강사 정보 관리, 전체 스케줄 조회·강사 할당, 공통/개인 영상 관리, 공지, 결제 내역, 이용 통계

3. 기능 요구사항

3.1 회원 관리 & 인증

  • [필수] 이메일·비밀번호 회원가입 / 로그인 / 비밀번호 찾기
  • [필수] 내 정보 수정, 프로필 이미지 등록
  • [필수] 회원 탈퇴 및 개인정보 처리방침 동의
  • [선택] 소셜 로그인 (Google, Apple) — 필요 시 협의

3.2 스케줄 예약 관리

  • [필수] 회원이 강사별 가능 시간대에서 1:1 수업을 직접 예약
  • [필수] 예약 변경·취소 (센터 정책 기반 가능 시간 제한)
  • [필수] 강사는 본인 스케줄 전체 조회, 예약 승인/거절
  • [필수] 관리자는 전체 예약 현황 조회 및 강사별 스케줄 할당

3.3 결제 (수강권)

  • [필수] 인앱 결제 — Google Play Billing (Android) / Apple IAP (iOS)
  • [필수] 수강권 구매·재연장 (회차/기간제)
  • [필수] 결제 영수증 서버 검증 및 수강권 자동 부여

3.4 운동 영상 콘텐츠

  • [필수] 공통 운동 영상 라이브러리 (카테고리/부위/난이도 필터)
  • [필수] 외부 유튜브 영상 링크 등록 및 앱 내 재생
  • [필수] 센터 자체 영상은 S3 + CloudFront HLS 스트리밍
  • [필수] 강사가 수업 후 개인 복습 영상을 촬영·업로드 → 해당 회원에게만 노출

3.5 1:1 채팅 (개인 메신저 대체)

  • [필수] 회원-강사 1:1 채팅 (WebSocket 실시간)
  • [필수] 텍스트, 이미지, 영상, 파일 전송 지원
  • [필수] 읽음 상태, 전송 상태 표시
  • [필수] 개인 전화번호·카카오톡 ID 등 비공개 — 앱 내 식별자만 사용

3.6 푸시 알림

  • [필수] 예약 확정/취소 알림
  • [필수] 수업 리마인더 (수업 전 N시간)
  • [필수] 채팅 메시지 수신 알림
  • [필수] 공지사항 등록 알림
  • [필수] 알림 수신 on/off 설정

3.7 관리자 페이지 (웹)

  • [필수] 회원 정보 통합 관리 (조회/수정, 수강권 잔여 확인)
  • [필수] 전체 예약 스케줄 관리 및 강사별 스케줄 할당
  • [필수] 공통/개인별 영상 콘텐츠 업로드 및 관리
  • [필수] 공지사항 등록 및 관리
  • [필수] 결제 내역 및 기본 서비스 이용 통계 (일/주/월 예약 수, 활성 회원 수, 매출)

3.8 데이터 마이그레이션

  • [필수] 기존 외부 솔루션 + 네이버 예약 회원 정보 → 신규 시스템 이전
  • [필수] 매핑 스크립트 작성 + 2회 이상 Dry-run 리허설
  • [필수] 이관 전/후 데이터 건수·키 무결성 검증 리포트 제공
  • [필수] 실패 시 롤백 절차 수립
